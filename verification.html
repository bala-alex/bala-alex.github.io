About software escrow verification

Though software developers deposit source code and other materials into escrow with best intentions, mistakes do happen. This is where technical verification of deposit materials comes into play.

Escrow materials can sometimes contain files or data that are corrupted, incomplete, password-protected, difficult to use, or even contain viruses. Documentation could be missing, passwords could be improperly documented, third-party dependencies not outlined or included, and the list goes on.

Verification is used to validate the completeness, accuracy, and usability of materials deposited into escrow. It’s the only way of being certain that, in case of a release event, the beneficiary will be able to effectively use their developer’s source code in-house or transition to a new supplier effortlessly.

Verification Micro

Our micro verification option includes an escrow content inspection report, including source code and any other data. This option also includes a review of development activity on the source code, and it’s a good starting point for companies that are simply looking to ensure that materials deposited in escrow are complete and up-to-date.

Verification Basic

Our basic verification option gives you a thorough code inspection, including source code, configuration materials, and other supporting documentation. This option is great for companies that want to validate the completeness and accuracy of escrowed materials to ensure they have everything they need to update the software application, without assistance from the licensor.

Verification Advanced

Our advanced verification option includes all the features of basic verification, in addition to installing your software application and running an operational version of the software. Companies for which it is imperative to ensure continuity in case of a release event often pick this option to validate not only the completeness and accuracy of materials deposited in escrow but also that the application runs as expected.

Verification Complete

Finally, our complete verification option includes all advanced verification features, plus testing up to 20 key features of your software. A feature can be anything contained in the source code; some examples of what this means include: the application sends an email confirmation, it has a working email signup process, the user is able to upload a file, and so on.

Though the goal here is not to conduct a full acceptance test of the software, this option is great for companies that want to minimize the chance of disruption to or delay in business operations by detecting any potential issues with critical features preemptively. It ensures that the beneficiary will be able to effectively use the application immediately and avoid any ramifications of losing access to mission-critical software for any amount of time.
